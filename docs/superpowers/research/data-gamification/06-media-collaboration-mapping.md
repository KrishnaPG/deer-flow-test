# Media And Collaboration Mapping

This file maps meetings, replays, charts, and rich media into RTS and RPG concepts.

## Collaboration And Media Objects

| Data Point | RTS Mapping | RPG Mapping | UI / Interaction Notes |
| --- | --- | --- | --- |
| Live meetings | War room, command bridge, council session | Campfire council, squad briefing, diplomatic audience | Show participants as seats / portraits with speaking activity |
| Meeting replays | Battle replay theater, debrief playback | Memory crystal replay, camp debrief recap | Replays should support timeline scrub, bookmarks, and chapter markers |
| Charts | Strategic telemetry dashboards, fleet economy graphs | Character stat charts, faction influence graphs | Charts should inherit the metaphor instead of looking like BI dashboards |
| Audio | Radio chatter, distress beacons, intercepted comms | Voice notes, companion whispers, ritual echoes | Keep transcript linked beside playback |
| Video | Recon feeds, drone camera footage, combat holograms | Vision replays, surveillance clips, flashbacks | Provide PiP and theater modes |
| Screenshares / live review streams | Sensor uplink, tactical holo-feed | Shared scrying view, mission visor link | Use presenter focus and participant reactions |
| Notifications | Fleet alerts, threat pings, command advisories | Quest alerts, companion interjections, omen warnings | Severity should control motion, sound, and persistence |
| In-game events | Sector invasions, breakthroughs, queue completions | Quest beats, ambushes, companion moments | Event feed should chain related events together |
| References discussed in meetings | Briefing attachments, tactical dossiers | Evidence packets, codex links | Meeting notes should preserve cited evidence |
| Insights raised in meetings | New doctrine proposal | New theory / lead / party plan | Promote accepted insights into persistent objects |

## Meeting Structure Mappings

| Meeting Element | RTS Mapping | RPG Mapping |
| --- | --- | --- |
| Host | Fleet admiral / mission commander | Party lead / ship captain |
| Participants | Division commanders, carriers, specialist wings | Companions, advisors, emissaries |
| Agenda | Mission objectives, logistics review, threat analysis | Quest objectives, loyalty issues, story decisions |
| Interruptions | Enemy contact, sensor alerts, fleet damage | Companion banter, urgency trigger, hostile encounter |
| Decisions | New build order, reroute, strike authorization | Dialogue choice, quest branch, companion assignment |
| Replay value | Post-battle analysis | Narrative recall and consequence tracking |

## Rich Data Types

| Data Type | RTS Best Metaphor | RPG Best Metaphor | Suggested UI Surface |
| --- | --- | --- | --- |
| Chart collections | Operations room wall displays | Codex analysis screens | Docked side panel with time controls |
| Audio collections | Signal archive | Memory archive | Playlist / channel browser |
| Video collections | Recon vault | Vision archive | Theater / gallery view |
| Topic clusters | Sector dossiers | Story arcs | Folder tree or constellation map |
| Reference bundles | Mission packs | Evidence bundles | Expandable stack cards |

## Recommended Experience Rules

- Live meetings should feel like active command spaces, not passive transcripts.
- Replays should be a first-class mechanic because they naturally fit both battle review and narrative recall.
- Charts should be attached to actors, sectors, missions, or companions so they stay contextual.
- Audio and video should always preserve provenance, timestamps, and who or what generated them.
- Notifications should collapse into event stories rather than an endless flat stream.
