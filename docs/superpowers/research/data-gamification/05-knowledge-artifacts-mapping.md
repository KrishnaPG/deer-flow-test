# Knowledge And Artifacts Mapping

This file maps the system's knowledge-producing outputs into RTS and RPG concepts.

## Knowledge Objects

| Data Point | RTS Mapping | RPG Mapping | UI / Interaction Notes |
| --- | --- | --- | --- |
| Artifacts produced | Buildings, modules, weapons, ship upgrades, relics | Loot, crafted gear, enchanted items, codex entries | Artifacts should be collectible, inspectable, and equippable |
| Knowledge graphs | Star map, territory graph, tech tree, hyperspace lane web | Codex web, clue board, conspiracy map, lore atlas | Knowledge graphs should support zoom, link reveal, and path highlighting |
| Research topics | Tech tree branches, doctrine tracks, research labs | Skill schools, disciplines, lore domains, quest themes | Topics should create visible capability unlocks |
| References | Intel dossiers, map annotations, sensor archives | Library shelves, witness notes, lore citations | References should feel like supporting evidence, not clutter |
| Insights | Strategic doctrines, battlefield discoveries | Epiphanies, revelations, companion realizations | Insights should trigger temporary highlights or new actions |
| Ideas | Build proposals, fleet doctrines, expansion plans | Dialogue ideas, quest plans, theory notes | Ideas should begin as drafts before being promoted |
| Concepts | Faction doctrines, strategic abstractions, design blueprints | Schools of thought, mythic concepts, philosophical stances | Concepts work best as high-level categories above individual artifacts |
| Summaries | Command briefings, mission synopses | Journal summaries, debriefs | Summaries should sit above raw detail and link downward |
| Derived datasets | Refined resources, processed intel, crafted subcomponents | Distilled reagents, decoded tomes, synthesis outputs | Clearly mark lineage from source to derivative |
| Drafts / versions | Prototype buildings, unfinished modules | Rough manuscripts, unstable enchantments | Versioning should feel like upgrade paths, not duplicate clutter |

## Artifact Families

| Artifact Family | RTS Fantasy | RPG Fantasy | Suggested Presentation |
| --- | --- | --- | --- |
| Text reports | Command briefings | Journal pages | Rich tooltip + expandable detail card |
| Code artifacts | Automated factories, fabrication scripts | Arcane runes, programmable spells | Show capability unlocks and dependencies |
| Datasets | Resource containers, sensor sweeps | Ingredient bundles, evidence kits | Show source, quality, and downstream use |
| Images | Tactical reconnaissance captures | Scene illustrations, investigation photos | Use gallery wall / scouting album metaphors |
| Audio | Distress signals, comm intercepts | Voice memories, companion recordings | Present as playable comm chips / memory crystals |
| Video | Recon footage, battle recordings | Vision echoes, mission recordings | Use replay theater or holo-projector framing |
| Models / plans | Ship blueprints, turret layouts | Gear schematics, ritual diagrams | Best shown with lineage and unlock tree |

## Progression Logic

| System Concept | RTS Mapping | RPG Mapping |
| --- | --- | --- |
| Promotion from raw data to artifact | Refining ore into weapons/modules | Crafting ingredients into usable gear |
| Promotion from artifact to knowledge | Installing module into doctrine tree | Equipping artifact to unlock story/skill branch |
| Promotion from insight to action | New strategic command becomes available | New dialogue / quest / skill option appears |
| Demotion / invalidation | Obsolete tech or sabotaged module | Cursed item, disproven clue, failed theory |

## Recommended Rules

- Not every artifact should be a "weapon" in RTS mode; some should be infrastructure, sensors, or doctrine modules.
- Not every artifact should be "loot" in RPG mode; some should be codex fragments, story keys, or companion gifts.
- Knowledge graphs should not be decorative. They should gate navigation, unlock actions, and reveal causal structure.
- References should always anchor an insight or concept so the player can trace where truth came from.
