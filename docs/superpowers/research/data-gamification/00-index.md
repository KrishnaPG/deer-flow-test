# Data Gamification Index

This folder maps agentic-AI system concepts into playable game metaphors.

File guide:

- `01-rts-mapping.md` - original RTS macro-paradigm notes
- `02-rpg-mapping.md` - original RPG micro-paradigm notes
- `03-mechanics-and-stress-tests.md` - stress test scenarios as gameplay events
- `04-runtime-operations-mapping.md` - sessions, orchestration, traces, metrics, queues
- `05-knowledge-artifacts-mapping.md` - artifacts, knowledge graphs, topics, references, insights
- `06-media-collaboration-mapping.md` - meetings, replays, audio, video, charts, live collaboration

Reading order:

1. Start with `01-rts-mapping.md` and `02-rpg-mapping.md` for the high-level fantasy.
2. Read `04-runtime-operations-mapping.md`, `05-knowledge-artifacts-mapping.md`, and `06-media-collaboration-mapping.md` for concrete data-to-game mappings.
3. Use `03-mechanics-and-stress-tests.md` to translate failure and scale events into gameplay and UI states.

Design rule:

- RTS mode should emphasize macro command, fleets, logistics, territory, production, and live telemetry.
- RPG mode should emphasize companions, dialogue, character sheets, quests, lore, morality, and tactical intervention.
- Both modes should present the same backend truth, only through different metaphor layers.
